Software

We present a fully automatic resistor sorter, with bee-hive inspired design elements. The active components of the resistor sorter consist of a modular conveyor belt, a two-pronged resistor probe, linearly-actuated “paddles”, and an array of bins. To sort the resistors, the resistors are first placed on the conveyor belt. Each of the conveyor belt modules has conductive copper tape on either end to act as conductive contacts with the resistors. When the conveyor belt moves, the modules pass under the two-pronged resistor probe, which comes into contact with the copper tape, passing current through the resistor. Using another resistor of a known value, we can determine the resistance of the resistor by measuring the voltage across the resistor. The resistor then passes to the underside of the conveyor belt, where the paddles knock the resistors into their respective bins. To stylize our resistor sorter like a beehive we rastered hexagonal motifs to the body of the sorter and added “melting honey” resin-printed accents, made one of the walls out of clear acrylic, and added LED lighting to the interior.

User Interface

User Interface To allow the user to control which resistors get sorted into each of XX bins, we use a touchscreen controlled by a Raspberry Pi. The UI depicts 12 boxes, each representing one bin, and uses a beehive-inspired color scheme to match our design goal. When the user clicks on a box, a number pad appears. Here the user can input a new resistance value (in Ohms) that they want to assign to the bin. Once they confirm, the Raspberry Pi updates a JSON file where all the bin states are stored for future use, and a signal is sent to the PCB by way of an Arduino Nano to update the PCB on the new bin state. We use a JSON to store the bin states as it allows us to maintain user preferences across multiple runs for convenience, storing the resistance values assigned to each bin during the prior use. We chose an Arduino Nano to bridge between the Raspberry Pi and the PCB because the Raspberry Pi operates on 3.3V, while the PCB operates on 5V. Connecting these two directly with a communication system like UART would fry the GPIO pins on the Raspberry Pi. Instead, we plug the Nano into the Raspberry Pi through the USB ports, and then connect the Nano to the PCB using its Serial pins. This prevents the pins on the Raspberry Pi from burning out and bypasses having to use a logic level converter or a similar voltage buffer that could impact signal integrity.
